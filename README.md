## 📸 Overview

The Enugu Smart City platform is a research prototype developed as part of an MSc thesis in Geographic Information Science at the University of Nigeria, Nsukka. It demonstrates the feasibility of a comprehensive smart city digital twin for a Nigerian city using exclusively open-source and freely available tools.

**Key capabilities:**

| Module | Description |
|--------|-------------|
| 🏢 **3D City Model** | 44,817 CityGML LOD1 buildings rendered in CesiumJS |
| 🌊 **Flood Risk** | HAND-based building-level risk classification (validated, 78% accuracy) |
| 🌧️ **Rainfall Simulation** | Real-time Rational Method flood inundation with animated building colours |
| 🚗 **Traffic Simulation** | SUMO microscopic simulation streamed live to the 3D scene via TraCI |
| 🌤️ **Live Weather** | OpenWeatherMap real-time rainfall coupled to flood simulation |
| 💨 **Air Quality** | Open-Meteo CAMS forecast data with EPA AQI index |
| 🚒 **Emergency Dispatch** | OSRM route optimisation with animated 3D vehicle tracking |
| ☀️ **Shadow Simulation** | 24-hour sun/shadow animation with adjustable time speed |

## ⚡ Quick Start (Windows)

### Prerequisites

| Software | Download | Notes |
|----------|----------|-------|
| **Python 3.9+** | [python.org](https://python.org/downloads) | ✅ Check "Add to PATH" during install |
| **SUMO 1.23+** | [sumo.dlr.de](https://sumo.dlr.de/docs/Downloads.php) | Installer sets `SUMO_HOME` automatically |

### 1 — Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/enugu-smart-city.git
cd enugu-smart-city
```

### 2 — Add the large data files

The following files exceed GitHub's 100 MB file limit and must be obtained separately:

- **`frontend/buildings_3dtiles/`** — 3D Tiles building model (~487 MB)
  Download from: [Google Drive link] or contact the author
- **`data/sumo/`** — SUMO road network files (~135 MB)
  Generate automatically by running `data/sumo/setup_sumo_enugu.bat`

### 3 — Launch

Double-click `START_ENUGU_SMARTCITY.bat`

The launcher will:
1. Install all Python packages automatically (first run only, ~2 min)
2. Build the SUMO network if not present
3. Start all four backend servers
4. Open the browser at `http://localhost:8080`

## 🔧 Manual Setup

If you prefer to start services manually:

```bash
# Install Python dependencies
pip install -r backend/requirements_flood.txt

# Run flood risk analysis (first time only, ~5-10 minutes)
python flood_analysis.py

# Start all backends in separate terminals:
python backend/app.py          # Routing API      → port 5000
python backend/flood_api.py    # Flood Risk API   → port 5001
python backend/traffic_api.py  # Traffic API      → port 5002

# Serve the frontend
cd frontend
python -m http.server 8080
```

Then open `http://localhost:8080` in Chrome or Firefox.

| Panel | How to activate |
|-------|-----------------|
| **Weather** | Loads automatically — add API key for live data |

1. Select a **FROM** station from the dropdown
2. Click anywhere on the map to set the **INCIDENT** location
3. Click **Dispatch** — a vehicle animates along the computed route
4. Click **✕ Clear Scene** after arrival to reset

## 🚗 SUMO Traffic Setup

SUMO traffic runs in **synthetic mode** by default (vehicles follow Enugu road geometry without real SUMO). To enable live SUMO simulation:

```bash
# Step 1: Build the Enugu road network (run once)
cd data/sumo
.\setup_sumo_enugu.bat

# Step 2: Enable SUMO in traffic_api.py
# Open backend/traffic_api.py and change line 44:
USE_SUMO = True   # was False

# Step 3: Restart traffic_api.py
python backend/traffic_api.py
```

The SUMO thread will load the 135 MB network (~20 s), then print:
```
[SUMO] TraCI connected ✓ — vehicles appear in ~5 seconds
```

## 📊 Flood Risk Methodology

The flood risk analysis uses the **Height Above the Nearest Drainage (HAND)** method (Nobre et al., 2011), which classifies buildings by their vertical distance from the nearest drainage channel:

| Risk Category | HAND Value | Return Period | Buildings |
|--------------|-----------|---------------|-----------|
| 🔴 High Risk | < 1.0 m | ~2-year flood | 2,847 (6.4%) |
| 🟠 Medium-High | 1.0 – 3.0 m | 5–10-year | 3,391 (7.6%) |
| 🟡 Medium | 3.0 – 6.0 m | 25-year | 5,208 (11.6%) |
| 🟢 Low Risk | 6.0 – 10.0 m | 100-year | 8,164 (18.2%) |
| 🔵 No Risk | ≥ 10.0 m | > 100-year | 25,207 (56.2%) |

**Validation**: 78% overall agreement with 3 documented Enugu flood events (2012, 2017, 2022). 83% agreement with Ejikeme et al. (2023) GPS field survey data.

To regenerate the flood risk data:
```bash
python flood_analysis.py
```
This takes approximately 5–10 minutes and produces `data/flood_risk_results/flood_risk_buildings.geojson`.

## 🔌 API Reference

### Flood Risk API (port 5001)

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/flood/statistics` | GET | Risk category counts and totals |
| `/api/flood/risk` | GET | All buildings with risk classification |
| `/api/flood/risk?lon=X&lat=Y` | GET | Nearest building risk at coordinate |

### Traffic API (port 5002)

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/traffic/health` | GET | Server status, mode, vehicle count |
| `/api/traffic/step` | GET | Current vehicle positions (lon, lat, heading, speed, type) |
| `/api/traffic/routes` | GET | Road network geometry (5,000 edges) |
| `/api/traffic/flood-adjust` | POST | Slow vehicles in flood conditions |

### Routing API (port 5000)

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/route` | POST | Compute emergency vehicle route |
| `/api/pois` | GET | Emergency service station locations |

## 🐛 Troubleshooting

**Buildings don't appear**
→ Check that `frontend/buildings_3dtiles/tileset.json` exists. This folder is not included in the repository due to file size.

**Flood colours don't show**
→ Run `python flood_analysis.py` first to generate the flood risk data. Check that `data/flood_risk_results/flood_risk_buildings.geojson` exists.

**Traffic shows SYNTHETIC not SUMO LIVE**
→ SUMO must be installed and `USE_SUMO = True` must be set in `backend/traffic_api.py`. The SUMO network must also be built first using `setup_sumo_enugu.bat`.

**"Array buffer allocation failed" crash**
→ This was fixed in v20 by switching to `GroundPolylinePrimitive`. Make sure you are using the latest version of `traffic_simulation.js`.

**Port already in use**
→ Run the launcher again — it clears ports 5000–5002 and 8080 automatically. Or restart your computer.

**PROJ/CRS errors in flood_analysis.py**
→ The script automatically sets `PROJ_DATA` to resolve conflicts between geospatial library installations. If errors persist, run: `pip install pyproj --upgrade`
